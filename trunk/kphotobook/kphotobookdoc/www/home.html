Overview

KPhotoBook is a powerful, easy to use, intuitive and fast application to manage and organize your photos. The main idea is to be able to annotate your photos with tags and to refind the photos using the given tags. The tags which can be created by yourself are arranged in a tree.

The KPhotoBook GUI comes up with several points which makes the handling with many photos very easy. This is just a short excerpt of it's goodies:

  • It is very easy to annotate tags to one or more images at a time
  • It is very easy to find images by the given tags (actually one mouse click only is needed)
  • It is very easy to edit the image database with an external program – it's saved as XML file

Please consider having a look at the handbook for further information.

Basic Features List

KPhotoBook will have at least the following features:

  • creating a photo database with photos from several folders which can be added single or recursively
  • annotate tags to the photos - this can be done for one photo only or for several photos at once
  • the tags are arranged in a tree
  • there will be different kind of tags: boolean, string, number
  • there can be written a description for each file
  • searching and filtering the photos using the given tags and description
  • all pictures matching the chosen filter are displayed as thumbnails
  • the pictures can be rotated (real, exif or database only)
  • showing the exif informations of a selected file
  • showing the pictures matching the filter in a slide show
  • opening the files with a any desired program

History

The idea for a photo database and photo albumprogram came up during September 2003. Having bought a digital cameraa few months ago, the necessity for a program to manage all the takenphotos became urgent.

At this time, we did not know about a comparable program for KDE. A few weeks later KimDaBa 1.0 was released.We looked into this program but decided to go on with the developmentof KPhotoBook anyway because KimDaBa was not exactlywhat we needed.

In February 2004 a first alpha-version of Fotoon was released. Fotoon came closer to what our idea is... butnot exactly. We decide – once again – to go on working onKPhotoBook and to have a close eye on these other applications.

Evolution of KPhotoBook

Version 0.0.3

Version 0.0.3 was the first release of KPhotoBook and was released in February 2004. It did not provide many features but seemed quite stable already (probably because of the lack of features :-) ). The UI provided a source directory tree and a main window in which thumbnails of the images in the imported directories were shown. There was also a tag tree - it was hard coded at this time. V0.0.3 already had the ability to alter the thumbnail size with a ruler.

Version 0.0.4

Version 0.0.4 was released in March 2004. Besides the features of Version 0.0.3, we provided a functional tag tree with title tags and boolean tags, improved filter abilities a few new icons and a new tool bar.

Why another application to manage photos?

It's not easy to manage hundreds or even thousandsof images. Other applications promote themselves to solve thisproblem. They say it'd be easy to manage such a huge amount ofphotos. Having looked at those other applications, we found that itwas not that easy and not nearly as straight forward as wethink such an application should be. Beside this, it is also veryimportant that it is not only easy to manage the photos but also tobe fast. Other applications provide the function to tag the picturesbut do not offer this function in a really easy and fast way. Someothers have the same approach as KPhotoBook, but waiting foran application to become usable when no work is done on it is notreally a great thing to do.
