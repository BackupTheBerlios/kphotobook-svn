Overview

KPhotoBook is a powerful, easy to use, intuitive and fast application to manage and organize your photos. The main idea is to be able to annotate your photos with tags and to refind the photos using the given tags. The tags which can be created by yourself are arranged in a tree.

The KPhotoBook GUI comes up with several points which makes the handling with many photos very easy. This is just a short excerpt of it's goodies:

  • It is very easy to annotate tags to one or more images at a time
  • It is very easy to find images by the given tags (actually one mouse click only is needed)
  • It is very easy to edit the image database with an external program – it's saved as XML file

Please consider having a look at the handbook for further information.

Basic Features List

KPhotoBook will have at least the following features:

  • creating a photo database with photos from several folders which can be added single or recursively
  • annotate tags to the photos - this can be done for one photo only or for several photos at once
  • the tags are arranged in a tree
  • there will be different kind of tags: boolean, string, number
  • there can be written a description for each file
  • searching and filtering the photos using the given tags and description
  • all pictures matching the chosen filter are displayed as thumbnails
  • the pictures can be rotated (real, exif or database only)
  • showing the exif informations of a selected file
  • showing the pictures matching the filter in a slide show
  • opening the files with a any desired program

History

The idea for a photo database and photo album program came up during September 2003. Having bought a digital camera few months ago, the necessity for a program to manage all the taken photos became urgent.

At this time, we did not know about a comparable program for KDE. A few weeks later KimDaBa 1.0 was released. We looked into this program but decided to go on with the development of KPhotoBook anyway because KimDaBa was not exactlywhat we needed.

In February 2004 a first alpha-version of Fotoon was released. Fotoon came closer to what our idea is... but not exactly. We decided – once again – to go on working on KPhotoBook and to have a close eye on these other applications.

Why another application to manage photos?

It's not easy to manage hundreds or even thousands of images. Other applications promote themselves to solve this problem. They say it'd be easy to manage such a huge amount of photos. Having looked at those other applications, we found that it was not that easy and not nearly as straight forward as we think such an application should be. Beside this, it is also very important that it is not only easy to manage the photos but also to be fast. Other applications provide the function to tag the pictures but do not offer this function in a real easy and fast way. Some others have the same approach as KPhotoBook, but waiting for an application to become usable when no work is done on it is not really a great thing to do.
