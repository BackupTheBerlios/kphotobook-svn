Changelog KPhotoBook

Version 0.0.7

Released in September 2005.

Known issues

  • The filter does not always work correct if the operator is 'or'.
  • After adding new patterns for files to handle you must restart the application. A simple rescan does not find the new file-types.

Before sending a bugreport or a wish you should consider the bug list and tasks on the KPhotoBook page on berlios.

Changes

  • Changed buildsystem to bksys/scons (see Links section)
  • A lot of minor and greater things 'behind the scenes'. Bug fixes and improvements.
  • Added tool tips to the tag tree
  • All icons are 'SVG' now
  • Bold tag tree node when selected images contain tag
  • Image importer
  • Internal image viewer
  • Menu clean-ups
  • More 'what's this' texts
  • New tag type: date time
  • New tag type: option list / option
  • New tag type: string
  • Showing untagged images when exiting
  • Splash screen
  • Tags can be set as 'secret'
  • Tip of the day

Version 0.0.6

Released in June 2004.

  • Tagging can be locked/unlocked now to avoid accidental change of tags. The lock can be unlocked temporary by holding the CTRL-key.
  • The tree states (open/closed) nodes and the filters are stored.
  • It is possible now to export selected/matching images as symbloc links to directory.
  • Menu entries for showing/restoring the toolviews. It does not exactly what I did want, but the MDI (Toolview) API is a bit odd. But I think it works well enough.
  • Add a quit action which closes all open window and exits KPhotoBook.
  • Install a mimetype for KPhotoBook associating '*.kpb' files with KPhotoBook.

Version 0.0.5

Released in April 2004. There are a lot of things to configure now and the GUI is cleaned up.

  • When setting filters on a parent tags, files associated with the sub tags are considered too. E.g. if dog is a sub tag of animal then, when searching for images containing an animal, the images tagged with dog must be displayed too.
  • GUI mode is changeable between IDEAl and TabPage mode.
  • Remember the selected files whenever possible (changing filter, changing preview size, ...).
  • Set F5 as shortcut for "refresh view" not for "rescan filesystem".
  • Ctrl+R as alternate shortcut for "refresh view".
  • Use right mouse button on tag filters to cycle backwards through the options.
  • The dock configuration is stored now. KDE is really great!
  • ...

Version 0.0.4

Version 0.0.4 was released in March 2004. Besides the features of Version 0.0.3, we provided a functional tag tree with title tags and boolean tags, improved filter abilities a few new icons and a new tool bar.

Version 0.0.3

Version 0.0.3 was the first release of KPhotoBook and was released in February 2004. It did not provide many features but seemed quite stable already (probably because of the lack of features :-) ). The UI provided a source directory tree and a main window in which thumbnails of the images in the imported directories were shown. There was also a tag tree - it was hard coded at this time. V0.0.3 already had the ability to alter the thumbnail size with a ruler.
