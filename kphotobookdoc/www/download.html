Download KPhotoBook

Downloads and cvs access are available on the projectsite at sourceforce.net: KPhotoBook at sourceforge.net

Building from CVS

Follow these instructions to build the application from the repository:

  • Login to the cvs-repository at sourceforge (no password is required): 
    cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/kphotobook login
  • Checkout the sourcecode: 
    cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/kphotobook co kphotobooksrc
  • Change into the checked out directory
    cd kphotobooksrc
  • Create the makefile: 
    make -f Makefile.cvs
  • Start configure (where KDE-installation-path is '/opt/kde3' for example): 
    ./configure --prefix='KDE-installation-path'
  • Build the application: 
    make
  • Change to root: 
    su
  • Install the application: 
    make install

Building from the tarball

Follow these instructions to build the application from the tarball:

  • Downloaded the tarball from KPhotoBook at sourceforge.net.
  • Extract the downloaded tarball: 
    tar -xzf kphotobook-x.x.x.tar.gz
  • Go into the extracted directory: 
    cd kphotobook-x.x.x
  • Start configure (where KDE-installation-path is '/opt/kde3' for example): 
    ./configure --prefix='KDE-installation-path'
  • Build the application: 
    make
  • Change to root: 
    su
  • Install the application: 
    make install

Gentoo installation

An ebuild request was submitted to gentoo bugzilla: #47560. Hopefully, it will soon be in the official portage tree.

For now, you can download the ebuild v0.0.6.
There are two ways to get it going:

  1. copy the ebuild into your portage tree (you will loose the ebuild after an rsync)
  2. use the portage overlay mechanism (preferred method, that way you won't loose the ebuild when updating the portage tree):
    Define 'PORTDIR_OVERLAY="/some/dir' in your make.conf file and save the ebuild as "/some/dir/media-gfx/kphotobook/kphotobook-VERSION.ebuild". The default location of the portage overlay dir is '/usr/local/portage'.
Either way, you need to generate a package manifest and digest file by changing to the kphotobook-ebuild-directory and running ebuild kphotobook-VERSION.ebuild digest.
Then the usual emerge kphotobook will do the job.
