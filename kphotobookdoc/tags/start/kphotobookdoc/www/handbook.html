'Database'

A KPhotoBook database is stored as one XML-file. This file contains all source-directories, tags and files belonging to this database. Of course it is possible to have several databases storing different types of photos (e.g. you could have a database for your private photos and a database for your business photos).

When a database has changed (tags or source-folders were edited) the file must be saved. That means that changes are not automatically persisted. Each time a file is saved the old file is backed-up.

Source Folders

A database can contain all images of one or more source folders. Source folders can be added non-recursively or recursively:

  • non-recursive: only the files in the selected folder will be added to the database.
  • recursive: all files in the selected folder and all sub-folders will be added to the database.

When adding a folder recursively hard- and soft-links are followed by default. To prevent endless-loops KPhotoBook tries to detect and suppress loops in the filesystem directory structure.

When restarting KPhotoBook every source folder will be scanned and new found sub-directories and files are addded to the database. This function can also be executed manually. Missed directories or files will not automatically be removed from the database to prevent loss of the association between files and tags.

Tags

Tags are used to describe what a photo contains or what properties it has. There are different kinds of tags:

  • title: this is a pseudo tag to group other tags together. It cannot be associated with a photo.
  • boolean: such a tag is associated with a photo or not.
  • enumeration: basically several enumerations are grouped together.
    Maximally one of the grouped entries can be selected (e.g. group: Quality contains the enumerations "high", "medium" and "low").
  • string: ??? how to describe?
  • numeric: ??? how to describe?

Each tag can contain a name and optionally an icon. The tags can be structured hierarchically that means you creat a tree of tags.

  • Quality

When a photo is associated with a tag it is implicitely associated with all parent tags as well. If you have a photo tagged with "Bob" and another photo tagged with "John", searching for photos containing "Persons" will find find these two photos.

Photo Manipulation

The only yet foreseen manipulation of the photos is rotating them. This can be done in three ways:

  • storing in database: The rotation information is stored in the database only.
  • altering EXIF: The EXIF header of the image is adjusted.
  • rotating physically: The image itself is rotated.
